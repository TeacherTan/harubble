# 界面设计说明

## 布局

当前主界面采用“左侧专辑导航 + 主内容区 + 底部播放器 Dock + 右侧按需滑出面板”的结构：

```text
┌──────────────┬──────────────────────────────────────────────┐
│ 专辑侧栏     │ 主内容区                                     │
│ AlbumCard ×N │ 顶部工具栏                                   │
│              │ 专辑舞台（封面大图）                         │
│              │ 专辑信息 / 操作按钮 / 曲目列表 / 内容滚动区  │
│              │                                              │
│              │ 底部播放器 Dock                              │
│              │ ├─ 传输控制 / 进度 / 乱序 / 循环 / 下载      │
│              │ └─ 歌词面板 / 播放队列面板                   │
└──────────────┴──────────────────────────────────────────────┘
                         ▲                    ▲
                         └── 设置 Sheet        └── 下载任务 Sheet
```

实现上，左侧和主内容区都使用 `OverlayScrollbars` 管理滚动，macOS 下顶部保留拖拽区域。

## 主要组件

当前主界面由以下组件拼装而成：

- `App.svelte`：应用入口，负责专辑加载、播放状态同步、下载任务状态同步、歌词、主题色、滚动舞台和各类面板编排
- `AlbumSidebar.svelte`：左侧专辑导航容器，内部渲染 `AlbumCard.svelte`
- `TopToolbar.svelte`：顶部工具栏，提供刷新、下载任务入口、设置入口和活动下载数量 badge
- `AlbumWorkspace.svelte`：主内容区容器，包裹专辑舞台、专辑信息和曲目列表滚动区
- `SongRow.svelte`：曲目行；默认点击播放，进入多选模式后改为点击勾选
- `PlayerDock.svelte`：底部 Dock 容器，内部承载 `AudioPlayer.svelte`
- `AudioPlayer.svelte`：播放器主体，包含播放控制、进度、乱序 / 循环、歌词切换、队列切换和当前歌曲下载入口
- `SettingsSheet.svelte`：右侧设置面板，负责下载参数、通知偏好和缓存清理
- `DownloadTasksSheet.svelte`：右侧下载任务面板，负责任务列表、进度、失败项、取消 / 重试和历史清理
- `StatusToastHost.svelte`：顶部 toast 宿主，用于替代同步 `alert()` 反馈

## 交互模式

### 顶部工具栏

当前顶部工具栏不再承担“player / download”双模式切换，而是只负责全局入口：

- 刷新当前缓存与页面数据
- 打开下载任务面板
- 打开设置面板
- 展示活动下载任务数量 badge

设置面板和下载任务面板互斥打开：打开一个时会关闭另一个。

### 曲目点击行为

当前实现采用“默认播放，按需进入多选”的模式：

- 默认状态下，点击 `SongRow` 会立即播放该曲，并使用当前专辑曲目构造播放上下文
- 点击曲目行右侧下载按钮，会为该曲创建单曲下载任务
- 进入多选模式后，点击 `SongRow` 不再播放，而是切换选中状态
- 多选模式下，曲目右侧下载按钮禁用，避免与批量下载语义冲突

### 多选下载

多选下载入口位于专辑信息区，而不是顶部工具栏。进入多选模式后，界面会显示：

- `多选下载 / 取消多选`
- `全选`
- `清空`
- `反选`
- `下载所选歌曲`
- 已选歌曲数量文案

批量下载最终调用 `create_download_job`，使用 `kind = selection` 创建任务。

## 播放状态流

前端通过 Tauri command 拉起播放，通过 Tauri event 持续同步状态：

```text
App.svelte
  ├─ invoke('play_song')
  ├─ invoke('pause_playback')
  ├─ invoke('resume_playback')
  ├─ invoke('seek_current_playback')
  ├─ invoke('play_next' / 'play_previous')
  └─ listen('player-state-changed' / 'player-progress')

src-tauri/src/main.rs
  └─ AudioPlayer
       ├─ cpal 输出
       ├─ symphonia 解码
       ├─ souvlaki 媒体会话
       └─ emit PlayerState
```

事件载荷统一是 `PlayerState`，前端在一次同步里可以拿到：

- 当前曲目
- 播放 / 暂停 / 加载状态
- 上一首 / 下一首是否可用
- 当前进度与总时长
- 当前音量
- 乱序和循环状态
- 当前播放队列与顺序

播放器 Dock 额外提供：

- 乱序开关
- 列表循环 / 单曲循环切换
- 歌词面板开关
- 播放队列面板开关
- 当前歌曲下载按钮

## 视觉与主题

当前前端采用新的设计 token 和玻璃拟态风格（glassmorphism）：

- 全局颜色、阴影、圆角和动效通过 CSS 变量与设计 token 驱动
- 顶部工具栏、播放器 Dock、设置 Sheet、下载任务 Sheet 使用半透明背景和模糊材质
- 专辑舞台使用大图封面、遮罩层和滚动联动的收缩 / 固化效果
- `prefers-reduced-motion` 会收敛动效时长与滚动行为

主题色仍然由当前专辑封面驱动：

- `extract_image_theme` 用于提取主题色并更新 UI 强调色
- 远程封面会优先通过 `get_image_data_url` 转成 data URL，再用于主舞台和播放器封面显示，降低跨域 / 远端图片兼容问题

## 下载相关 UI

### 设置面板（SettingsSheet）

右侧设置面板当前负责：

- 输出格式选择：`wav` / `flac` / `mp3`
- 输出目录选择：通过 `@tauri-apps/plugin-dialog` 选择本地目录
- 歌词 sidecar 开关：有歌词时写入同名 `.lrc`
- 下载完成通知开关
- 播放切换通知开关
- 测试通知发送
- 清理播放缓存：调用 `clear_audio_cache`

### 下载入口

当前下载能力分成三条链路：

1. 播放器下载按钮：调用 `create_download_job`（`kind = song`），下载当前播放曲目
2. 专辑页“下载整张专辑”按钮：调用 `create_download_job`（`kind = album`），一键整专下载
3. 专辑页多选下载：在多选模式下批量勾选歌曲，点击“下载所选歌曲”调用 `create_download_job`（`kind = selection`）

当相同歌曲、专辑或同一组选择已存在运行中 / 排队中任务时，UI 会用 toast 给出“已在队列中”的提示。

### 下载任务面板（DownloadTasksSheet）

工具栏下载图标会打开独立下载任务面板。该面板当前可展示：

- 任务标题、任务类型、状态标签
- 总进度与进度文案
- 任务级错误摘要
- 子任务（单曲）级状态
- 子任务错误信息
- 任务取消 / 重试
- 子任务取消 / 重试
- 历史清理

空状态下会提示用户从“下载整张专辑”或曲目下载按钮开始创建任务。

## 反馈与通知

当前前端不再优先使用阻塞式 `alert()`，而是通过 `StatusToastHost.svelte` 挂载的 toast 系统给出反馈，包括：

- 创建下载任务成功 / 已在队列中
- 清理下载历史结果
- 清理音频缓存结果
- 发送测试通知结果
- 各类失败提示

系统通知开关由设置面板控制，具体平台行为仍以 Tauri 通知插件和操作系统权限结果为准。

## 文档与实现的当前关系

本文件以当前 `main` 分支代码为准。与仓库早期说明相比，当前实现已经确认以下变化：

- 顶部工具栏不再提供全局 `player / download` 双模式切换
- 多选下载入口从顶部工具栏收回到专辑信息区
- 下载设置和下载任务都改为右侧独立 Sheet，而不是固定第三栏
- 播放器已经演进为带歌词、队列、乱序 / 循环和下载入口的底部 Dock
- 用户反馈以 toast 为主，不再依赖同步阻塞弹窗
