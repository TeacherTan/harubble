# 后端路线图

> 记录后端各阶段的完成状态与未来增强方向。若与最新实现冲突，以代码为准。

## 总览

**已完成：**

- Phase 1–10
- Phase 12A（库内搜索 MVP）
- Phase 12B 首批搜索增强
- 首页模块（后端数据层 + 前端视图）
- Tag Registry 自定义元数据系统
- Tag Editor 双层编辑与三路合并
- 合集系统（官方合集 + 用户合集 CRUD + 导入导出）

**待办：**

- Phase 11（条件触发）：历史视图后端支撑
- Phase 12B 剩余：召回质量、排序策略、中文输入体验
- Phase 12C：歌词全文检索

> Phase 11 属于条件触发型，搜索能力先行进入 Phase 12，不按编号严格顺序。

## 待办阶段

### Phase 11：历史视图后端支撑（条件触发）

触发条件（满足任一才进入）：

1. 历史记录规模明显增长
2. 前端基于完整快照的筛选出现性能问题
3. 历史视图需要分页或惰性加载

方向：列表与详情分离、摘要查询、渐进增强。

### Phase 12B 剩余：召回与排序

方向：

- 拼音召回增强
- 标题前缀 / 连续子串的更细排序
- 短 query 阈值控制
- n-gram 兜底召回

### Phase 12C：歌词全文检索

前置项：歌词 cache owner、状态模型、刷新策略、隐私边界。

方向：

- `LibrarySearchScope` 扩展为包含 `lyrics`
- 歌词字段索引与短片段 preview
- 缓存刷新后的索引更新
